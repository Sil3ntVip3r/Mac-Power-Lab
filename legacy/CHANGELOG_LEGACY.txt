MacPowerLab v0.8.6
====================

Reporting-focused upgrade for the MacBook power monitor and stress-test suite.

NO SSD STRESS TESTS ARE INCLUDED.

Main additions in v0.4.0:
- Automatic self-contained HTML report generator
- Charger/cable scorecard
- Charger verdict: keeps up / not keeping up / partial
- Battery health report
- Charge curve analysis by battery percentage band
- Event timeline with plain-English explanations
- Port-controller warning checks
- External watt-meter readings
- Test metadata capture for charger/cable/port comparisons
- Compare multiple logs
- Long-term history database
- Monitor version bumped to v0.11.0

v0.5.1 ENERGY MODE INTERPRETATION FIX
-------------------------------------

This patch fixes the macOS 27 Energy Mode interpretation.

Your Battery settings screenshot shows:
- On battery: High Power
- On power adapter: High Power

But system_profiler may still expose LowPowerMode=Yes at the same time.
v0.5.1 now derives the active mode this way:

1. If HighPowerMode=Yes, show High Power and do not warn.
2. Else if LowPowerMode=Yes, show Low Power and warn.
3. Else show Normal / Automatic.

The raw System Information fields are still shown in --full for debugging.


v0.5.2 ENERGY MODE FALSE-POSITIVE FIX
-------------------------------------

macOS 27 may report raw LowPowerMode=Yes even when the Battery settings screen
shows High Power.

v0.5.2 changes the warning logic:

- HighPowerMode=Yes -> High Power, no warning.
- LowPowerMode=Yes + HighPowerMode explicitly No/False/Off -> Low Power warning.
- LowPowerMode=Yes + HighPowerMode missing/unknown -> Unknown / raw LowPowerMode=Yes, no warning.

This prevents the main dashboard from showing a false Low Power Mode warning when
the user-facing macOS Battery settings are set to High Power.

v0.8.6 AC CHARGER LOAD ESTIMATE FIX
------------------------------------

Built after reviewing a v0.8.4 monitor-only upload.

Findings:
- Monitor ran correctly.
- CPU cluster summary worked.
- macOS thermal pressure showed Nominal.
- Battery charged from 77% to 100%.
- But charger load showed 100% for the whole run.

Cause:
- The monitor treated adapter_calculated_watts / 28V * 5A = 140W as actual live draw.
- That value is normally the negotiated USB-C PD capacity/contract, not measured draw.

Fix:
- On AC Power, charger load now estimates adapter output as:
    charging:       primary system load + battery charge acceptance
    battery assist: primary system load - battery discharge assist
    neutral:        primary system load
- visible_load_estimate_w now uses this AC output estimate.
- charger_load_percent = AC output estimate / adapter rated watts.
- Adds:
    ac_output_estimate_w
    ac_output_estimate_source
    battery_assist_w
- UI adds "Load source" so the estimate is transparent.

Note:
- Exact wall draw still requires a wall or USB-C watt meter.
